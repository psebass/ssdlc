# Bienvenido al Camino del Desarrollo Seguro

Una guía práctica y estratégica sobre DevSecOps.  
Aquí encontrarás metodologías, análisis de vulnerabilidades CWE y las mejores prácticas para desplazar la seguridad hacia el origen del desarrollo (_Shift Left_).


## Introducción

El SSDLC (Secure Software Development Life Cycle) bajo la filosofía DevSecOps busca guiar al lector a través de metodologías integrales que convergen en un objetivo crítico: desplazar la seguridad hacia la izquierda (Shift Left). 
Este enfoque propone invertir mayor dedicación en la seguridad desde las etapas tempranas del ciclo de vida de desarrollo, reduciendo riesgos y costos de remediación en producción.

## Glosario
Análisis Dinámico Automatizado (DAST)  
Técnica que evalúa la seguridad de una aplicación mientras está en ejecución, identificando vulnerabilidades mediante simulaciones de ataques sin acceder al código fuente.

Análisis Estático de Seguridad (SAST)  
proceso que examina el código fuente en reposo para detectar fallas de seguridad, funciones inseguras y malas prácticas antes de ejecutar la aplicación.

Autenticación multifactor (MFA)  
Método de verificación que combina dos o más factores (contraseña, token, biometría) para asegurar que solo usuarios legítimos accedan al sistema.

Cifrado débil  
Uso de algoritmos obsoletos o configuraciones inseguras que pueden ser vulneradas con facilidad, comprometiendo
datos sensibles almacenados o transmitidos.

Control de acceso  
Conjunto de reglas que determinan qué usuarios pueden ver o ejecutar determinados recursos, evitando accesos y acciones no autorizadas.

Inyección SQL  
Vulnerabilidad donde datos no validados alteran una consulta SQL, permitiendo acceder, modificar o eliminar información en la base de datos.

Modelado de amenazas (STRIDE vs DREAD)  
Metodología para identificar riesgos analizando vectores de ataque y su impacto potencial mediante clasificadores como STRIDE o escalas como DREAD.

OWASP Proactive Controls  
Lista de prácticas recomendadas para que desarrolladores integren seguridad desde el inicio, como validar datos, proteger identidades y manejar errores adecuadamente.

Sesiones y tokens seguros  
Conjunto de prácticas para proteger identificadores de sesión y tokens de autenticación mediante expiración controlada, cookies seguras y mecanismos antifraude.

Uso de criptografía para proteger datos  
Empleo de algoritmos modernos como AES, RSA o SHA-256 para garantizar la confidencialidad, integridad y autenticidad
de la información en tránsito o en reposo.
